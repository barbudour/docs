SpeShu.AI — это маркетплейс AI-моделей. Мы приглашаем провайдеров подключать свои модели и инфраструктуру к нашей платформе, чтобы сделать их доступными для разработчиков, компаний и корпоративных клиентов.

## Преимущества партнёрства

**Доступ к аудитории.** Тысячи разработчиков и компаний используют SpeShu.AI для работы с нейросетями и AI-инструментами.

**Монетизация.** Вы получаете доход от каждого запроса к вашим моделям.

**Инфраструктура.** Мы берём на себя биллинг, мониторинг, маршрутизацию запросов и поддержку пользователей.

**Простая интеграция.** Подключение строится через стандартный OpenAI-совместимый API, поэтому интеграция не требует сложной доработки с вашей стороны.

## Как стать провайдером

Сначала вы оставляете заявку и рассказываете о своих моделях, инфраструктуре и технических возможностях.

После этого команда SpeShu.AI проводит техническую проверку: оценивает совместимость API, стабильность работы и качество моделей.

Затем мы подключаем ваш API к платформе SpeShu.AI и вместе тестируем интеграцию на реальном трафике.

После успешного тестирования ваши модели становятся доступны пользователям SpeShu.AI.

## Технические требования

### Совместимость API

Ваш API должен быть совместим с OpenAI Chat Completions API.

**Пример запроса:**

```bash theme={null}
POST /v1/chat/completions
Content-Type: application/json
Authorization: Bearer <API_KEY>

{
  "model": "your-model-name",
  "messages": [
    {"role": "user", "content": "Hello!"}
  ]
}
```

### Обязательные поля ответа

Ответ должен содержать стандартные поля, совместимые с OpenAI-форматом.

**Пример ответа:**

```json theme={null}
{
  "id": "chatcmpl-xxx",
  "object": "chat.completion",
  "created": 1234567890,
  "model": "your-model-name",
  "choices": [
    {
      "index": 0,
      "message": {
        "role": "assistant",
        "content": "Response text"
      },
      "finish_reason": "stop"
    }
  ],
  "usage": {
    "prompt_tokens": 10,
    "completion_tokens": 20,
    "total_tokens": 30
  }
}
```

### Дополнительные требования

| Требование | Описание |
| --- | --- |
| Uptime | Минимальная доступность сервиса — 99,5% |
| Latency | TTFB меньше 2 секунд для большинства запросов |
| Rate limits | Инфраструктура должна выдерживать минимум 100 RPS |
| Streaming | Поддержка SSE для потоковой генерации |
| Мониторинг | Метрики для отслеживания качества и стабильности |

Чтобы стать провайдером SpeShu.AI, свяжитесь с нами через страницу поддержки и отправьте информацию о ваших моделях.

**Что указать в заявке:**

* Название компании.
* Список моделей, которые вы хотите подключить.
* Примерные объёмы: RPS, токены в день, ожидаемая нагрузка.
* Техническую документацию по API, если она есть.
* Контакты технического специалиста.
