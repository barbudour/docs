  <Card title="AI-менеджер по продажам" icon="briefcase">
    **Задача:** оптимизировать работу менеджеров, которые тратят время на заполнение CRM и формулировку ответов на возражения.

    **Решение:** SpeShu.AI API подключён к CRM и базе продуктов. По параметрам клиента собирает персонализированное КП, после звонка заполняет карточку сделки, по запросу формулирует ответ на конкретное возражение.

    **Результат:** менеджер обрабатывает вдвое больше заявок в день.
  </Card>

  <Card title="AI-консультант по маркетингу" icon="palette">
    **Задача:** оптимизировать работу маркетолога, который тратит дни на подготовку текстов, баннеров и адаптацию креативов под форматы.

    **Решение:** SpeShu.AI API получает параметры акции или продукта и возвращает тексты для всех каналов, визуалы в нужных форматах и варианты для A/B-теста.

    **Результат:** цикл подготовки кампании сокращается с трёх дней до одного. Команда занимается стратегией, а не производством.
  </Card>

  <Card title="AI-аналитик отчётов" icon="chart-bar">
    **Задача:** наладить передачу данных и систематизировать массивы.

    **Решение:** данные из таблиц и систем поступают в SpeShu.AI API, модель пишет аналитический отчёт, выделяет отклонения и формулирует выводы.

    **Результат:** отчёт готов автоматически. Руководитель читает выводы, а не собирает цифры.
  </Card>

  <Card title="AI-продюсер видео" icon="video">
    **Задача:** оптимизировать расходы на фотосессии, съёмки и монтаж.

    **Решение:** SpeShu.AI API принимает фото продукта или референс, генерирует видео через Kling или Seedance, адаптирует под нужные форматы и соотношения сторон.

    **Результат:** видео для любого канала — без съёмочной группы и монтажёра, за несколько минут.
  </Card>

  <Card title="AI-ревьюер кода" icon="microchip">
    **Задача:** ускорить code review и быстрее выпускать продукты.

    **Решение:** SpeShu.AI API подключён в CI. При каждом PR проверяет код на ошибки и стиль, генерирует unit-тесты для новых функций, обновляет документацию по изменённым модулям.

    **Результат:** первый проход ревью происходит до того, как коллега открыл PR. Технический долг перестаёт копиться.
  </Card>

  <Card title="AI-рекрутер" icon="user-check">
    **Задача:** регулярно фильтровать кандидатов и выбирать 10 претендентов на каждую позицию.

    **Решение:** каждое резюме поступает в SpeShu.AI API, модель оценивает по заданным критериям, пишет сводку и ставит оценку соответствия. Для финалистов составляет вопросы под конкретного кандидата.

    **Результат:** шортлист готов на следующий день. HR сразу работает с людьми, а не с документами.
  </Card>

  <Card title="AI-бухгалтер" icon="calculator">

    **Результат:** три дня первичной обработки превращаются в полдня. Бухгалтер работает с исключениями, а не с потоком бумаг.

## FAQ

<AccordionGroup>
  <Accordion title="Какие модели доступны через API?">
    Прямо сейчас можно использовать порядка 50 моделей. Текстовые ChatGPT, Claude, Gemini, DeepSeek и ещё 8 нейросетей. Для генерации изображений подойдут ChatGPT Image, Seedream, Nano Banana и другие. Генерировать видео можно в WAN, Grok, Kling и Seedance.
  </Accordion>

  <Accordion title="Можно ли платить от юридического лица?">
    Да, официально работаем с юридическими лицами. Выставляем счёт, принимаем оплату по СБП или другим удобным способом, фиксируем в договоре для бухгалтерии.
  </Accordion>

  <Accordion title="Есть ли закрывающие документы?">
    Есть. Акты и счета формируются автоматически. Поддерживаем ЭДО — можно настроить обмен документами через вашу систему.
  </Accordion>

  <Accordion title="Можно ли ограничить расходы?">
    Да. Для каждого сотрудника или проекта можно выставить дневной и месячный лимит. Расходы видны в реальном времени с детализацией по моделям.
  </Accordion>

  <Accordion title="Есть ли тестовый доступ?">
    Мы можем выделить от 1 до 10 миллионов токенов на первичное тестирование. Их стоимость засчитается при заключении договора. Общая сумма к оплате снизится.
  </Accordion>

  <Accordion title="Можно ли подключить API к CRM/боту/сайту?">
    Да. API совместим со стандартом OpenAI, поэтому встраивается в большинство готовых решений без переписывания кода. Если нужна помощь с интеграцией, наша поддержка отвечает на все вопросы и консультирует.
  </Accordion>

  <Accordion title="Где хранятся данные?">
    На серверах в России. Данные не используются для обучения моделей.
  </Accordion>

  <Accordion title="Есть ли поддержка внедрения?">
    Да. После подключения помогаем с первым запросом, подбираем подходящие модели под вашу задачу и остаёмся на связи при масштабировании.
  </Accordion>

  <Accordion title="Можно ли использовать несколько моделей в одном проекте?">
    Да, и это одно из главных преимуществ SpeShu.AI. Благодаря единому API-ключу с одним балансом пользователь может переключаться между моделями внутри одного проекта без дополнительных настроек.
  </Accordion>

  <Accordion title="Чем SpeShu.AI отличается от прямого подключения к OpenAI/Claude/Gemini?">
    Три разных провайдера — это три аккаунта, три способа оплаты в иностранной валюте, три набора документов и три точки отказа. SpeShu.AI заменяет всё это одним API-ключом, одним балансом в рублях и одним личным кабинетом с контролем расходов. Плюс вы получаете доступ к моделям, которых нет напрямую в России.
  </Accordion>
</AccordionGroup>
